Software Documentation

Medical Labelling Management Software

Tools used: Microsoft Word, Snagit, Adobe Photoshop

This project brief involved producing a series of user manuals from a variety of sources including technical specifications and SME notes. These manuals were requested to accompany a global brand of medical labelling software.

Ongoing projects required updating and restructuring manuals in response to new software releases and additional functionality. With each new release, a new approach was taken to improve the usability of documentation.

Antonym Banking Software

Tools used: Adobe Acrobat, Snagit, Adobe Photoshop

This project involved creating Installation and SDK documentation for a sophisticated interbank lending product. The sources for the content were documentation created by developers. This documentation needed to be revamped in order to meet the needs of implementation consultants.

Work entailed collating all the existing information, organizing it, creating a consistent format and style and then rewriting the content. New staff needed to be able to use the documentation in order to assist with implementation under minimal supervision.

Zed Attack Proxy - Web Application Penetration Testing tool

Tools used: Googledocs, Github, Atom, Snagit, Adobe Photoshop, Articulate

This project involved working with a remote team of designers and developers to create all written content for the ZAP Website. The target audience were specialists in the field of web application penetration testing at various levels.

Content included website copy in a marketing style, quick start guides, videos, tutorials, curated content from an existing wiki and content sourced from interviewing SMEs.
