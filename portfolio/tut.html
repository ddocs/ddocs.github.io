Zed Attack Proxy - Web Application Penetration Testing tool

Tools used: Git, Github, Markdown, Atom

This project involved creating content for a new ZAP website. ZAP is a Web Application Penetration Testing Tool

The approach I took was to start by understanding ZAP users especially the needs of people new to security testing. This would form the basis of the content strategy.

I researched, identified and interviewed target users. After identifying three key audiences, I created user personas and stories for them, created a content strategy and began crafting content.

Web Application Penetration Testing is a very specialist area so it was important that I was able to obtain accurate source material. To do this, I interviewed SMEs as much as possible - Developers, QA Testers and Penetration Testers. I also collated existing content from a variety of sources, identifying gaps and asking lots of questions.

Content was authored in markdown using git commands to collaborate and manage content in GitHub.
